Notifications (ntfy.sh)
Run the test script first:
python test_ntfy.py
  • If it says latin-1 codec error — replace tracker.py, msrp_alert.py with latest versions
  • If it says Notification sent successfully but phone shows nothing — open the ntfy app, confirm your topic is subscribed, and check phone Settings → Apps → ntfy → Notifications are Allowed
  • If it says Connection error — check your internet connection
  • If it says HTTP 4xx — your topic name has invalid characters (only letters, numbers, hyphens, underscores allowed)
The topic name in tracker.py must match EXACTLY what you subscribed to in the ntfy app, including capitalization.
Most common causes:
  • Phone battery saver killed the ntfy app in the background — go to phone Settings → Battery → ntfy → set to Unrestricted
  • ntfy app was updated and needs notification permission re-granted
  • Your ntfy topic subscription expired (free topics are permanent, but check the app)
Run python test_ntfy.py to confirm the sending side is working, then focus on the phone side.

Binder & Data Persistence
The binder saves to your browser's localStorage. This can be lost if:
  • You cleared your browser history or site data
  • You opened the binder in a different browser (localStorage is browser-specific)
  • You used a private/incognito window (localStorage doesn't persist there)
Always open the dashboard from the same browser at http://localhost:8080/binder.html — not from a different browser or incognito mode.
There is currently no export/backup feature for binder data. If this is a concern, let Claude know and a CSV export can be added.

Open data/price_history.db in DB Browser for SQLite (free at sqlitebrowser.org) then use the Execute SQL tab: 
-- Products ever seen at or below MSRP
SELECT name, retailer, price_str, recorded_at
FROM price_records
WHERE pct_of_msrp <= 100
ORDER BY recorded_at DESC;

-- Lowest price ever seen per product
SELECT name, retailer, MIN(price) AS lowest, msrp
FROM price_records
GROUP BY url ORDER BY lowest;

-- How often each product was in stock
SELECT name, retailer,
  SUM(in_stock) AS times_in_stock,
  COUNT(*) AS total_checks
FROM price_records GROUP BY url;

-- Price trend for one product (last 7 days)
SELECT recorded_at, price_str, in_stock
FROM price_records
WHERE name LIKE '%Chaos Rising ETB%'
  AND retailer = 'target'
ORDER BY recorded_at DESC LIMIT 168;
With 52 products logged hourly, the database grows at approximately:
  • 52 records/hour x 24 hours = ~1,250 records/day
  • ~37,500 records over 30 days (display window)
  • ~112,500 records over 90 days (retention window)
At roughly 200 bytes per record, the database will stay under 25MB after 90 days. Records older than 90 days are purged automatically on every hourly log cycle. Run python plugins/price_history.py at any time to see the current size.
Amazon Monitor
This is intentional for your safety as a collector. Pokemon TCG products from third-party Amazon sellers carry documented authenticity risks including resealed packs and weighted boxes. Amazon.com as the direct seller guarantees the product came from an authorized distributor and is covered by Amazon's A-to-Z guarantee if anything is wrong.
  • Amazon.com sold - urgent alert + browser opens + add to cart attempted
  • FBA (Fulfilled by Amazon) - lower priority alert only, flagged as "verify authenticity" - no automatic browser or cart action
  • Third-party marketplace - not alerted at all
Amazon MSRP windows are rare and brief. The monitor checks every 15 minutes specifically to catch these windows without putting heavy load on your system.
Open plugins/amazon_monitor.py and find the AMAZON_PRODUCTS list near the top. Add a new entry in the format: 
("Product Name", "ASIN", None)
The ASIN is the 10-character code in the Amazon URL: amazon.com/dp/B0GR6N18F6. Use None for the third value and the MSRP will be determined automatically from the product name. Run python plugins/amazon_monitor.py to check the current status of all monitored products.
  • Run the standalone diagnostic: python plugins/amazon_monitor.py - this prints the current price, seller, and availability for all watched products
  • Make sure you are logged into Amazon in the browser profile: python plugins/cart_preloader.py --setup --retailer amazon
  • Amazon occasionally serves captcha pages to unrecognized browsers - logging in via the setup command helps establish a legitimate session
  • Check data/amazon_monitor_history.json to see when products were last checked and what prices were recorded
Best Buy Invite Monitor
Best Buy uses a two-phase system for high-demand Pokemon TCG products:
  • Phase 1 - Request Invite: When the "Request Invite" button appears on a product page, the monitor detects it, sends you an ntfy alert, opens a browser, and automatically clicks the button to register your interest as early as possible.
  • Phase 2 - Selected: When Best Buy selects you, the monitor detects your status change, sends an URGENT alert, and opens a browser directly to checkout. You have 24 hours to complete the purchase - you click Place Order.
Early registration likely improves your selection odds since Best Buy's algorithm considers account activity and purchase history.
The script never clicks Place Order. You must complete the purchase yourself within the 24-hour window.
Run the standalone diagnostic: 
python plugins/bestbuy_invites.py
This checks all tracked Best Buy products and prints the current invite state for each: UNAVAILABLE, REQUESTABLE, REQUESTED, or SELECTED. Also check data/bestbuy_invite_history.json to see timestamps for when each state was detected.
Walmart Queue Monitor
Four types of Walmart activity are monitored:
  • Wednesday drops - sends an alert at 11:45 AM ET on Wednesdays (15 minutes before the public 12 PM drop window) so you have time to get to your computer
  • Any-day restocks - when any tracked Walmart product flips in-stock via the main tracker, a dedicated Walmart alert fires and a browser opens to that product page
  • Clearance and rollbacks - scans at 9 AM and 6 PM daily for clearance badge, rollback pricing, or more than 10% below MSRP
  • New listings - scans at 7 AM and 1 PM for untracked Pokemon TCG products appearing at or near MSRP
Check data/walmart_drop_history.json to see all previous alerts and detected drops.

Store Inventory Checker
The store inventory checker runs automatically at 8 AM daily. To run it immediately: 
python plugins/store_inventory.py
It will ask for your zip code, then scan the nearest Target and Walmart stores and tell you which specific store has stock, what aisle, and how many units. Results are also saved to data/store_inventory.json.
Best days to run an in-store check are Tuesday and Wednesday when vendor routes refill shelves. Mid-morning (9-11 AM) typically has the freshest stock.
